Semana 1

Para entender el desarrollo web, primero debemos diferenciarlo del diseño web, aunque ambos términos a menudo se confunden. El diseño web se refiere a la apariencia y la percepción del sitio web, abarcando elementos como el diseño gráfico y el logotipo. Está más relacionado con la estética y la experiencia del usuario que con las funciones del sitio. Los diseñadores web se encargan de que el sitio sea visualmente atractivo y fácil de usar, ajustándose al propósito para el que fue creado.

El desarrollo web, en cambio, se enfoca en las funcionalidades y características del sitio. Incluye la programación de la "parte trasera" del sitio, cubriendo funciones como el registro de usuarios, los sistemas de gestión de contenido, el comercio electrónico y cualquier aplicación basada en bases de datos. Los desarrolladores web hacen posible que los visitantes interactúen con el sitio de manera efectiva, asegurando que las diversas funciones operen correctamente. Es decir, mientras el diseño web se preocupa por cómo se ve el sitio, el desarrollo web se preocupa por cómo funciona.

Antes de adentrarnos en las herramientas de desarrollo web, es crucial comprender la arquitectura cliente-servidor. Esta arquitectura implica que un cliente (generalmente un navegador web) realiza peticiones a un servidor, que luego responde a estas solicitudes. Aunque esta idea puede aplicarse a programas que se ejecutan en una sola computadora, resulta más ventajosa en sistemas operativos multiusuario distribuidos a través de una red de computadoras. Los servidores manejan múltiples solicitudes concurrentes y garantizan la seguridad del sistema, mientras que los clientes son programas que participan activamente en la comunicación, enviando peticiones y esperando respuestas.

En el ámbito del desarrollo web, el trabajo solía recaer en un único rol conocido como Webmaster. Hoy en día, sin embargo, el desarrollo web se caracteriza por la integración de diversos roles especializados, cuyo trabajo en conjunto es crucial para el éxito del proyecto. Estos roles incluyen desarrolladores frontend, diseñadores de interfaces, maquetadores, desarrolladores backend, administradores de bases de datos, gestores de proyectos y testers, entre otros. Cada uno de estos roles se encarga de diferentes aspectos del desarrollo web, desde la lógica del negocio y la programación del servidor hasta el diseño visual y la experiencia del usuario. Las tecnologías clave utilizadas en este campo son HTML, CSS y JavaScript, que permiten a los desarrolladores estructurar, estilizar y añadir funcionalidad a las páginas web.

Semana 2

En una página web es fundamentalmente un documento de texto con marcas o etiquetas que permiten modificar su presentación, incluir elementos multimedia, crear hiperenlaces y añadir significado al texto. Estas etiquetas se escriben entre desigualdades (como <p>, <h1>,<div>) y suelen ir por parejas, con una etiqueta de apertura y una de cierre. Algunas etiquetas, conocidas como marcas vacías (como <img>, <br>, <hr>), no requieren una etiqueta de cierre. Los navegadores web interpretan estas etiquetas y presentan el documento según el diseño del autor, ignorando en general los saltos de línea, espacios en blanco y tabuladores, excepto en la etiqueta <pre>.

Es una buena práctica no utilizar etiquetas HTML para modificar el aspecto visual del contenido, sino delegar este trabajo a las hojas de estilo (CSS). Las etiquetas pueden anidarse, pero no de cualquier manera; algunas pueden estar contenidas por otras, pero no se pueden insertar en cualquier orden. Los atributos dentro de las etiquetas permiten especificar características adicionales y pueden tener valores. Por ejemplo, el atributo "id" debe ser único en toda la página, mientras que el atributo "class" permite definir clases de elementos que se pueden referenciar en las hojas de estilo.

Un documento HTML está estructurado en dos partes principales: la cabecera ( <head>) y el cuerpo ( <body>). La cabecera contiene información de control e identificación, como el título de la página y enlaces a hojas de estilo o scripts, mientras que el cuerpo contiene el contenido visible en el navegador. La primera etiqueta del documento, , define el tipo de documento y su versión, siendo "html" la correspondiente a HTML5. Las etiquetas básicas incluyen <html> para englobar todo el documento, <head> para la cabecera y <body> para el contenido principal.

En cuanto a formularios, que permiten la interacción del usuario con el servidor, se delimitan con la etiqueta <form>. Los atributos más importantes de esta etiqueta son "action", que especifica el agente que procesará los datos enviados, y "method", que define cómo se enviarán los datos (get o post). Los controles dentro de los formularios, como <imput>, <button>,<select>, <textarea>, permiten al usuario introducir datos. Cada tipo de control tiene atributos específicos, como "type" para definir el tipo de dato (texto, número, contraseña), "name" para identificar el control, y "value" para establecer su valor inicial.

Semana 3

CSS (Hojas de Estilo en Cascada) es una tecnología que permite separar el contenido de una página web del diseño visual. En lugar de incluir tanto el contenido como el formato en el HTML, CSS permite definir el estilo de presentación en archivos separados. Esto facilita la creación de diseños consistentes y su modificación en un solo lugar, lo que es especialmente útil cuando se aplican estilos a múltiples páginas web.

Los estilos en CSS se pueden aplicar de tres maneras: usando el atributo <style> directamente en el HTML, con la etiqueta <style> dentro del HTML, o mediante una hoja de estilos externa vinculada con <link>. Las hojas de estilo externas son las más recomendables para mantener un diseño coherente en múltiples páginas, ya que cualquier cambio en la hoja de estilo se refleja en todas las páginas que la utilizan.

Los selectores en CSS permiten especificar qué elementos deben recibir un estilo particular. Estos incluyen selectores de tipo (<E>), clase (<.nombre_clase>), identificador (<#nombre-id>), y combinaciones de elementos como descendientes (<E> <F>), hijos (<E> > <F>), y elementos consecutivos (<E> + <F>). Cada tipo de selector tiene su uso específico y puede combinarse para aplicar estilos de manera precisa.

La aplicación de estilos en CSS sigue unas reglas de precedencia: los estilos en línea tienen mayor prioridad sobre los internos y externos, mientras que los estilos en hojas externas tienen la menor prioridad. Además, el navegador aplica las propiedades de mayor especificidad en caso de conflicto. Esto permite un control detallado sobre cómo se presentan los elementos en una página web.

Semana 4

En este módulo se revisa cómo incluir JavaScript en una página web. El código JavaScript puede ser incluido directamente en la etiqueta <script> del HTML, que se ejecuta automáticamente al procesar la etiqueta. Alternativamente, se puede utilizar un archivo externo vinculado a través del atributo src de <script>, facilitando la organización y mantenimiento del código. El archivo puede estar en una ruta absoluta, relativa, o en una URL externa.

La ubicación de la etiqueta <script> dentro del HTML también afecta el rendimiento, y puede colocarse en el <head> o <body> del documento. JavaScript permite escribir sentencias, que se separan con puntos y comas, y agregar comentarios para documentar el código. Las funciones en JavaScript ayudan a evitar la duplicación de código y se pueden definir para realizar tareas específicas. Los parámetros pueden ser pasados a las funciones para personalizar su comportamiento.

Además, JavaScript usa el DOM (Document Object Model) para manipular elementos HTML, permitiendo la creación, modificación y eliminación de nodos. Los comentarios son útiles para explicar el código y no afectan su ejecución. JSON es un formato de datos basado en texto que puede ser utilizado para transmitir información y es convertido entre texto y objetos JavaScript mediante los métodos JSON.parse() y JSON.stringify().

Finalmente, HTTP es el protocolo que permite la comunicación entre clientes y servidores, con métodos como GET y POST para realizar solicitudes y enviar datos. AJAX permite realizar pedidos al servidor de forma asíncrona sin refrescar la página, utilizando la función fetch() para manejar las respuestas
